## 🔹 STEP 5: Compare Logic

```java
return asc ? a > b : a < b;
```

### 💡 Meaning:

| Order      | Condition       |
| ---------- | --------------- |
| Ascending  | swap if p1 > p2 |

## 🔹 STEP 6: Swap

```java

    packages.set(j + 1, p1);

```

# 🧠 Paano gumagana ang bubble sort (actual concept)

## 🔥 IDEA:

> Pinapa-"bubble" pataas ang pinakamalaking value bawat pass

---

## 🔹 Example (weight sorting ascending)

Initial:

```
[15, 12, 8]
```

---

### 🔁 PASS 1:

Compare:

```
15 vs 12 → swap → [12, 15, 8]
15 vs 8  → swap → [12, 8, 15]
```

👉 Largest (15) nasa dulo na

---

### 🔁 PASS 2:

```
12 vs 8 → swap → [8, 12, 15]
```

---

### ✅ Final:

```
[8, 12, 15]
```

---

# 🔄 Paano ito nangyayari sa code mo

## Iteration mapping:

| Code        | Meaning          |
| ----------- | ---------------- |
| `i` loop    | number of passes |
| `j` loop    | comparison       |
| `get(j)`    | current          |
| `get(j+1)`  | next             |
| `if (swap)` | decision         |
| `set()`     | actual swap      |

# ⚡ IMPORTANT LINE (core ng bubble sort mo)

```java
for (int j = 0; j < packages.size() - i - 1; j++)
```

### 💡 Bakit may `- i - 1`?

👉 Dahil:

* bawat pass → may isang element na sorted na sa dulo
* hindi na siya kailangan i-compare ulit
